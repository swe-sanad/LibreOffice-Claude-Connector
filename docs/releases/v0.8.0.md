# v0.8.0 — MCP server: 61 → 137 tools

This release completes the full [`docs/TOOLS-WANTED.md`](../TOOLS-WANTED.md) roadmap: a
prioritized, domain-by-domain wish-list of **85 proposed tools** (from a 5-expert gap
analysis of the LibreOffice UNO API against the prior 61-tool surface) is now
**essentially shipped as 76 new tools**, taking the stdio MCP server from **61 to 137
tools** across documents, Calc, Writer, and a set of cross-cutting umbrellas. See
[`docs/MCP-TOOLS.md`](../MCP-TOOLS.md) for the full generated catalog (137 tools, 16
sections). This is a **server-only** release — the `.oxt` extension is unchanged.

## Highlights

**Calc**
- `calc_create_pivot` / `calc_refresh_pivot` — build and refresh DataPilot pivot tables
  (row/column/page/data fields, aggregation functions).
- `calc_edit_chart` / `calc_list_charts` — modify an existing embedded chart's title,
  axis titles, legend, chart type, series colors, and data labels; enumerate charts.
- `calc_autofilter` / `calc_standard_filter` — turn on AutoFilter dropdowns or apply a
  multi-condition filter.
- `calc_named_ranges`, `calc_page_setup` / `calc_set_print_area` — named ranges and
  print layout.
- `calc_format_cells_advanced`, `calc_get_cell_format`, `calc_get_conditional_formats`,
  `calc_get_validation` — read back formatting/CF/validation state instead of only
  writing it.
- `calc_add_sparkline`, `calc_add_scale_format`, `calc_goal_seek`,
  `calc_multiple_operations`, `calc_remove_duplicates`, `calc_transpose`,
  `calc_group_outline`, `calc_group_shapes`, `calc_copy_sheet`.
- `calc_sort_range`, `calc_set_dimensions`, `calc_set_visibility`, `calc_move_sheet`,
  `calc_recalculate` — the good-first single-API wrappers.

**Writer**
- `writer_insert_toc` / `writer_update_indexes` — insert a Table of Contents and
  refresh all TOCs/fields after edits.
- `writer_bookmarks` — full bookmark lifecycle (list/insert/delete/get/set) as one
  umbrella tool.
- `writer_insert_cross_reference`, `writer_insert_footnote`, `writer_insert_shape`,
  `writer_insert_text_frame` — cross-references, footnotes/endnotes, draw shapes, and
  floating text frames.
- `writer_track_changes`, `writer_redact`, `writer_set_watermark`,
  `writer_set_page_background` — document review and presentation controls.
- `writer_edit_table`, `writer_set_image_layout`, `writer_add_section` — richer table
  editing, image anchoring/wrap, and multi-column/protected sections.
- `writer_mail_merge`, `writer_spellcheck`, `writer_apply_list`.

**Cross-cutting umbrellas**
- `set_hyperlink` — one tool for both Calc cells and Writer text.
- `export_document` — PDF (page range, PDF/A, password, quality) and CSV
  (delimiter/quote) export from one place.
- `list_styles` / `set_style` — discover and create/modify named styles across
  paragraph/character/cell/page/frame families.
- `protect_document` — Calc sheet/workbook and Writer section protection.
- `dispatch_uno` — escape hatch to run any `.uno:` command against the active frame.
- `document_undo`, `bind_document_event`, `set_view_zoom`, `get_signatures`,
  `list_embedded_objects`, `insert_ole_object`.

## Umbrella consolidation

Per `TOOLS-WANTED.md`'s "don't build two of these" guidance, overlapping proposals were
merged into one consolidated tool rather than built twice. **Skipped** in favor of the
umbrella: `calc_set_hyperlink`, `writer_insert_hyperlink` (→ `set_hyperlink`);
`writer_export_pdf` (→ `export_document`); `calc_define_name` (→ `calc_named_ranges`);
`writer_insert_bookmark` (→ `writer_bookmarks`); `writer_manage_styles` (→
`list_styles`+`set_style`); `calc_protect_sheet` (→ `protect_document` +
`calc_cell_protection`); `refresh_fields` (covered by `writer_update_indexes` +
`calc_recalculate`); `writer_insert_ole_chart` (→ `insert_ole_object object=chart`).

## ⚠️ Known limitations

No live-office exercise was possible this session — LibreOffice was in use by another
session at the time. The following version-sensitive tools are implemented
**best-effort**: `calc_create_pivot`, `calc_add_scale_format`, `calc_add_sparkline`,
`calc_multiple_operations`, `writer_mail_merge`. Their underlying UNO APIs vary across
LibreOffice versions, so on an incompatible API shape they are designed to **fail
in-band with a clear error message rather than crash the server** — but this behavior,
and the tools' happy paths, still need a live smoke-test against a running office before
being fully relied upon.
